Reality check

What this does — and what it can't

Everything runs in your browser. Nothing is uploaded to a server, which is exactly what makes it free and GitHub‑Pages friendly. That same constraint sets the limits below.

✓ Works fully, free, in‑browser

  • Spin‑the‑wheel word picker with selectable fields (Healthcare is the default).
  • Definitions — shown from a built‑in glossary, so they work offline / on GitHub Pages with no API key.
  • Prompt ideas generated from rule‑based templates around your word + field.
  • 5‑minute writing timer with a last‑minute red blink and a synthesized per‑second tick cue.
  • Recording of microphone, webcam, or both, at 5 / 10 / 20 / 30 minutes, with automatic stop and a mirrored self‑view for facial‑cue practice.
  • Live transcript via the browser's built‑in speech recognition (the same free tech behind voice typing), plus an optional on‑device Whisper second pass for a far more complete, timestamped transcript — all in your browser, nothing uploaded.
  • Pitch, volume and pace tracking charted against time, plus a downloadable recording + transcript.

✗ Not possible (or limited) on a free static host

  • True AI‑generated ideas/definitions need an LLM API key. A key can't be safely embedded in a public GitHub Pages site (anyone could read it), so ideas here are template‑based. To get real AI generation you'd add a tiny backend or serverless function — that's no longer purely static/free.
  • The actual Family Feud sound is copyrighted, so the countdown uses an original synthesized tick in the same spirit.
  • The live transcript isn't perfect. The browser's free speech engine works best in desktop Chrome/Edge (poor/absent in Firefox/Safari) and can miss words. For a near‑complete version, use the Accurate transcript button on the report — it runs Whisper on‑device over your saved audio. The trade‑off: a one‑time model download (~40–80 MB) and a processing wait (roughly 1–3 min for a 5‑min clip on "Tiny", longer on "Base"), because GitHub Pages can't send the headers that unlock multi‑threaded/fast WASM. It also may be blocked inside the in‑chat preview — run it on the deployed site.
  • Speech recognition listens to your microphone only — so "Webcam only" (no mic) records video but produces no transcript or pitch chart.
  • Webcam capture needs a camera and HTTPS; the recorded file is normal (un‑mirrored), while your live self‑view is mirrored like a mirror or video call.
  • "Tone" is measured objectively as pitch + loudness + pacing. Emotional/sentiment tone would need an ML model and isn't included.
  • Files stay on your device. There's no cloud save — you download the recording and transcript yourself.

Hosting on GitHub Pages

  • Put this file in a repo as index.html.
  • Repo → Settings → Pages → Source: main branch, root.
  • Open the https://<you>.github.io/<repo>/ URL over HTTPS — microphone and camera only work on HTTPS (GitHub Pages is HTTPS by default).
